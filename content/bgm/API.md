# BGM API

This API exposes the generated ScreenCam BGM catalog for preview and download.

## Endpoint

```http
GET /api/bgm.json
```

Production URL:

```txt
https://thescreen.cam/api/bgm.json
```

The response is static JSON generated by `npm run bgm:build`.

## Response

```ts
type BgmManifest = {
  version: 1;
  generatedAt: string;
  previewSeconds: number;
  tracks: BgmTrack[];
  tags: string[];
  keywords: string[];
};

type BgmTrack = {
  id: string;
  title: string;
  tags: string[];
  keywords: string[];
  duration: number;
  durationMs: number;
  size: number;
  sourceSha256: string;
  audioUrl: string;
  previewUrl: string;
  downloadUrl: string;
  preview: {
    start: number;
    seconds: number;
    size: number;
  };
};
```

## Field Notes

- `version`: manifest schema version. Currently always `1`.
- `generatedAt`: ISO timestamp from the newest source BGM file mtime used by the generator.
- `previewSeconds`: target preview length in seconds.
- `tracks`: full BGM catalog.
- `tags`: unique tags across all tracks.
- `keywords`: unique keywords across all tracks.
- `id`: stable kebab-case track identifier.
- `title`: display title.
- `duration`: full track duration in seconds.
- `durationMs`: full track duration in milliseconds.
- `size`: full track file size in bytes.
- `sourceSha256`: SHA-256 hash of the source audio file.
- `audioUrl`: public full-track URL.
- `previewUrl`: public 5 second preview URL.
- `downloadUrl`: public download URL. Currently the same as `audioUrl`.
- `preview.start`: preview start offset in the full track, in seconds.
- `preview.seconds`: actual preview duration in seconds.
- `preview.size`: preview file size in bytes.

## Client Usage

```js
const response = await fetch("/api/bgm.json");
const manifest = await response.json();

for (const track of manifest.tracks) {
  console.log(track.title, track.previewUrl, track.downloadUrl);
}
```

Preview playback:

```html
<audio controls src="/bgm/previews/lofi-hiphop-80s-1-preview.mp3"></audio>
```

Download:

```html
<a href="/bgm/audio/lofi-hiphop-80s-1.mp3" download>Download</a>
```

## Caching

`/api/bgm.json` is prerendered and returned with:

```http
Cache-Control: public, max-age=3600
Content-Type: application/json; charset=utf-8
```

Static audio files are served from `public/bgm/`.

## Updating The API Output

1. Edit `content/bgm/tracks.json`.
2. Add source audio under `content/bgm/audio/`.
3. Run:

```bash
npm run bgm:build
```

Use full regeneration when needed:

```bash
npm run bgm:build:force
```
